IMPORTANT NOTE:
Normally, we wouldn't put env files in repositories. I wanted to share an example, so I renamed mine and added some notes.
Env files (short for environment) hold "environment variables" for your application. Therese can be thought of as top level global variables, which might be changed in deployment.
To use this env file. Please rename it (this file) to the following name exactly:
.env.local

Note that this filename should start with a period. That dot is part of the name. It should be in the same directory as this file, which is the same directory as the package.json.
You can also create a user by opening up a mongodb shell (mongosh) in compass and running the below as a command, replacing the password to whatever you want it to be:

db.createUser({
  user: "ReactConnectionLive",
  pwd: "whatever_password_you_want", 
  roles: [{ role: "readWrite", db: "react_class_demo_live" }]
});

Then, please place whatever password you chose into the line below where "YourPasswordHere" is located.
After that, please delete all lines in this file except those two. After that, you'll have a proper .env file and a mongodb username/password pair.


MONGODB_URI=mongodb://ReactConnectionLive:YourPasswordHere@localhost:27017/react_class_demo_live?authSource=react_class_demo_live
MONGODB_DB_NAME=react_class_demo_live
